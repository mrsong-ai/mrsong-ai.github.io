# 🔐 Pi Network 安全配置指南

## ⚠️ 重要安全提醒

**你的原始API密钥已经在GitHub上公开暴露！必须立即更换！**

## 🚨 立即行动清单

### 1. 更换所有Pi Network密钥

#### 📍 登录Pi Network开发者控制台：
- 访问：https://develop.pi
- 使用你的Pi账户登录

#### 🔄 重新生成密钥：
1. **API Key** - 在开发者控制台中重新生成
2. **验证密钥** - 重新生成验证密钥
3. **App ID** - 无法更改，但要确保安全使用

### 2. 更新代码中的配置

#### 📝 在以下位置更新新密钥：

1. **前端配置** (`qianduan/index.html`)：
   ```javascript
   appId: "YOUR_NEW_APP_ID_HERE"
   ```

2. **后端配置** (`houduan/lib/pi-network.js`)：
   ```javascript
   this.apiKey = process.env.PI_API_KEY || 'YOUR_NEW_API_KEY_HERE';
   this.appId = process.env.PI_APP_ID || 'YOUR_NEW_APP_ID_HERE';
   ```

3. **环境变量** (创建 `.env.local` 文件)：
   ```
   PI_API_KEY=你的新API密钥
   PI_APP_ID=你的App ID
   PI_VALIDATION_KEY=你的新验证密钥
   ```

### 3. 验证密钥配置

#### 🔍 如果需要验证密钥文件：
1. 创建 `qianduan/.well-known/validation-key.txt`
2. 将新的验证密钥放入此文件
3. **注意**：此文件会被公开访问，这是Pi Network的要求

## 🛡️ 安全最佳实践

### ✅ 应该做的：
- 使用环境变量存储敏感信息
- 定期更换API密钥
- 监控API使用情况
- 使用HTTPS部署

### ❌ 不应该做的：
- 将密钥直接写在代码中
- 提交 `.env` 文件到Git
- 在公开仓库中存储敏感信息
- 与他人分享API密钥

## 🔧 环境变量使用方法

### 开发环境：
1. 复制 `.env.local.example` 为 `.env.local`
2. 填入真实的密钥值
3. 重启开发服务器

### 生产环境：
1. 在部署平台设置环境变量
2. 不要在代码中硬编码密钥
3. 使用平台的密钥管理功能

## 📞 如需帮助

如果在配置过程中遇到问题：
1. 检查Pi Network开发者文档
2. 确认密钥格式正确
3. 验证网络连接和权限设置
